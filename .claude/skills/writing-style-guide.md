# 写作风格检测与细纲驱动写作使用说明

## 新增组件总览

| 组件 | 类型 | 用途 |
|------|------|------|
| `writing-style-checker` | Agent | 检测禁止句式、禁止词语和写作要求合规性 |
| `webnovel-outline-write` | Skill | 基于细纲驱动的章节写作 |
| `细纲模板.md` | 模板 | 细纲文件格式规范 |

---

## 一、writing-style-checker Agent

### 功能

检测章节内容是否违反写作要求，包括：

1. **禁止比喻词语**：火山、心湖、石子、湖面、针刺、羽毛、重锤、掀起、巨浪、泛起、涟漪、惊雷、某种、水中、波澜、水滴、炸弹
2. **禁止情绪暗示**：眼神/眼底闪过/闪烁、带着 xx 意味、带着某种、并没有立刻
3. **禁止状态描述**：不容拒绝、不易察觉、难以察觉、不可置信、不容置疑
4. **禁止动物比喻**：像一只 XX
5. **禁止平衡句式**：不是…而是…、不仅…更是…
6. **禁止总结性表达**：这个动作、这句话
7. **禁止生理细节**：指节、嘴角弧度、审视

### 调用方式

**自动调用**（集成在 webnovel-write 流程中）：
```
Step 4: 润色
    ↓
Step 4.5: writing-style-checker 检测
    ↓
Step 5: Data Agent
```

**手动调用**：
```bash
# 无直接手动命令，通过 webnovel-write 流程自动执行
```

### 检测结果处理

- **通过**：进入下一步
- **不通过**：返回 Step 4 重新润色，修复问题后重新检测

## 二、webnovel-outline-write Skill

### 功能

严格按照细纲生成章节内容，适用于：
- 关键章节
- 伏笔章
- 需要精确控制剧情走向的章节

### 调用命令

```bash
/webnovel-outline-write {chapter_num}
```

### 前置条件

1. **细纲文件必须存在**：`大纲/第{NNNN}章 -细纲.md`
2. **世界书/设定集可用**
3. **前文最新章节可用**

### 执行流程

```
Step 0: 预检（校验细纲/设定/前文）
    ↓
Step 1: 读取细纲与世界书
    ↓
Step 2: 按节拍生成正文（6 个剧情节拍）
    ↓
Step 3: 写作风格自检测
    ↓
Step 4: writing-style-checker 审查
    ↓
Step 5: 输出章节
    ↓
Step 6: Data Agent 回写状态
```

### 细纲剧情节拍格式

```markdown
1. XX 回到 XX（此处适当描写场景）
2. 众人在 XX 讨论
3. XX 意识到 XX
4. XX 开始调查 XX（通过 XX）
5. XX 发现 XX
6. XX 质问 XX（此处渲染 XX 情绪）

---
**截止于此完结**
```

### 输出要求

- **字数**：2300-3000 字（或细纲指定）
- **视角切换**：不超过 2 次，切换时需标注"主视角切回到 XX"
- **场景描写**：精简，不单独存在
- **创作重心**：对话、剧情推进、角色塑造

## 三、细纲模板使用

### 文件位置

`大纲/细纲模板.md`

### 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 章号 | 四位章号 | 0011 |
| 标题 | 章节标题 | 真相一角 |
| 字数目标 | 目标字数范围 | 2300-3000 |
| 主视角 | 本章主视角角色 | 林越 |
| 场景 | 主要场景位置 | 森林住所 |
| 登场角色 | 出场角色列表 | 林越、幽夜 |

### 剧情节拍说明

1. **XX 回到 XX**：场景引入，建立情境
2. **众人在 XX 讨论**：对话推进剧情
3. **XX 意识到 XX**：角色认知转变
4. **XX 开始调查 XX**：角色采取行动
5. **XX 发现 XX**：信息揭示，转折
6. **XX 质问 XX**：冲突高潮，情绪渲染

### 设定约束

**不可变更事实**：必须遵守的设定，如角色能力、世界观规则等

**禁止事项**：
- 不添加世界书未提及的机制
- 不添加前文未出现的角色（除非必要）
- 不解释角色行为动机

## 四、webnovel-write 更新

### 新增 Step 4.5

在原有 Step 4（润色）后新增 Step 4.5（写作风格检测）：

```
原流程：
Step 3: 审查 → Step 4: 润色 → Step 5: Data Agent

新流程：
Step 3: 审查 → Step 4: 润色 → Step 4.5: 写作风格检测 → Step 5: Data Agent
```

### 充分性闸门更新

新增条件：
- **Step 4.5 写作风格检测通过（无 high 级别问题）**

### 检测清单

Step 4.5 执行时会检查：
- [ ] 无比喻句（火山/心湖/石子等）
- [ ] 无情绪暗示（闪过/闪烁/带着）
- [ ] 无惰性状态描述（不容/不易/难以）
- [ ] 无动物比喻（像一只）
- [ ] 无平衡句式（不是…而是…）
- [ ] 无总结性名词化（这个动作/这句话）
- [ ] 无眼神/眼底描写
- [ ] 无嘴角/指节描写
- [ ] 少用介词连词副词

## 五、使用示例

### 示例：创作第 11 章

**步骤 1：创建细纲**

使用 `大纲/细纲模板.md` 创建 `大纲/第 0011 章 -细纲.md`：

```markdown
# 第 0011 章 细纲

## 章节信息
- 章号：0011
- 标题：真相一角
- 字数目标：2300-3000
- 主视角：林越
- 场景：森林住所
- 登场角色：林越、幽夜

## 剧情节拍
1. 林越回到住所（描写深夜森林的寂静）
2. 林越与幽夜讨论今天遇到的猎杀者
3. 林越意识到猎杀者公会已经盯上自己
4. 林越开始调查猎杀者公会的底细（通过询问幽夜）
5. 林越发现猎杀者公会在寻找纯血人族
6. 林越质问幽夜是否早就知道这件事（渲染被隐瞒的情绪）

---
**截止于此完结**
```

**步骤 2：调用 skill**

```bash
/webnovel-outline-write 11
```

**步骤 3：等待执行**

skill 会自动：
1. 读取细纲
2. 按节拍生成正文
3. 执行写作风格自检测
4. 调用 writing-style-checker 审查
5. 输出章节
6. 回写状态

**步骤 4：检查结果**

检查输出文件：
- `正文/第 0011 章.md`
- `.webnovel/summaries/ch0011.md`
- `.webnovel/state.json`

## 六、常见问题

### Q1: 细纲驱动和普通写作有什么区别？

**A**:
- `webnovel-outline-write`：严格按照细纲节拍生成，创作自由度低，适合关键章节
- `webnovel-write`：基于 Context Agent 任务书，有一定创作自由度，适合常规章节

### Q2: 写作风格检测不通过怎么办？

**A**:
1. 查看检测报告中的问题列表
2. 返回 Step 4 润色，修复问题
3. 重新执行检测

### Q3: 如何跳过写作风格检测？

**A**:
- 暂不支持跳过
- 如确需跳过，使用 `--minimal` 模式（仅跳过部分审查，不跳过风格检测）

### Q4: 细纲中可以添加新角色吗？

**A**:
- 不建议
- 如确需添加，确保不与世界观冲突
- 在"设定约束"中说明
