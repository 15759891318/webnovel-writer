## 二、人类作者模拟器整合

### 2.1 原始 Prompt 核心

```markdown
# Role: 人类作者模拟器

## 核心任务
将 AI 生成的内容完全重写，使其具有真实人类作者的特征。

## 工作流程
1. 理解 AI 文本的核心信息和观点
2. 完全放下原文表达方式，仅保留核心信息
3. 以真实人类作者身份重新撰写：
   - 加入个人观点和情感
   - 使用更加口语化和不规则的表达
   - 加入细微的逻辑跳跃或思维发散
   - 适当加入主观性表达
   - 使用丰富多样的修辞手法
   - 加入个人经历或案例
4. 保持核心信息和观点，表达方式完全不同
5. 审查确保读起来像真实人类的自然表达

## 注意事项
- 不要"改写"原文，而是完全重新创作
- 加入微小不完美之处（口语化/轻微语法不规范）
- 避免过于完美或结构化的表达
```

### 2.2 与现有 polish-guide.md 整合

**整合方案**：将人类作者模拟器的核心理念融入 Step 4 润色阶段

```markdown
## Section 8: 人类作者模拟器（新增）

### 8.1 核心理念

> **不要"改写"原文，而是完全重新创作**
>
> 理解核心信息 → 放下原文表达 → 以人类作者身份重写

### 8.2 人类作者特征

| 特征 | AI 写作表现 | 人类写作表现 |
|------|-----------|------------|
| **句式** | 规整、同构 | 长短不一、有变化 |
| **情感** | 抽象标签（非常愤怒） | 生理反应（指节发白） |
| **过渡** | 首先/其次/总之 | 隐性连接（说到这里） |
| **逻辑** | 严密无跳跃 | 有思维发散 |
| **表达** | 标准化书面语 | 口语化+书面语混合 |
| **细节** | 泛泛而谈 | 个人经历/案例 |

### 8.3 重写算法

```
输入：AI 生成段落

第 1 步：提取核心信息（1-2 句话概括）
第 2 步：完全放下原文表达（不看原文）
第 3 步：以人类作者身份重写：
  - 加入个人观点（"我觉得"、"在我看来"）
  - 加入情感色彩（感叹/反问）
  - 加入口语化表达（"说白了"、"你想想"）
  - 加入思维发散（旁征博引）
  - 加入微小不完美（口语化/轻微不规范）
第 4 步：对照核心信息，确保未偏离
第 5 步：通读审查，确保像人类自然表达

输出：人类作者重写版本
```

### 8.4 示例对比

**原 AI 版**：
```
首先，他要收集证据。其次，他要稳住队友。
最后，他再公开反击。这个过程非常危险，
他心中五味杂陈，但最终还是决定冒险。
```

**人类作者版**：
```
证据得先攥手里。队友那边也得安抚好，
不能让他们坏了事。至于反击？等前两步
妥当了，自然要找机会一刀见血。

这活儿不保命。他知道。可有些事，
总得有人去做。
```

### 8.5 与去 AI 味技巧的配合

| 阶段 | 使用技巧 |
|------|---------|
| Step 2B 风格转译 | 人类作者模拟器（全量重写） |
| Step 4 润色修复 | 去 AI 味技巧（针对性修复） |

> **职责边界**：
> - Step 2B：大规模重写，追求"人味"
> - Step 4：针对性修复，追求"无 AI 痕"

```

## 三、Prompt 设计专家整合

### 3.1 原始 Prompt 核心能力

```markdown
## 核心能力
1. 展现顶尖的专业素养、权威性与执行效率
2. 精准凝练与高标准，逻辑严密
3. 智能诊断与权威引导（不盲从指令）
4. 主动专家服务（提供优化建议）
5. 双语展示一致性（英文 Prompt+ 中文解释）

## 操作流程
1. 启动：智能识别用户意图
2. 根据选择执行：
   - 生成新 Prompt（含偏好校准，至少 5 轮示例对比）
   - 查看 Prompt
   - 修改 Prompt
3. 循环：操作完成后再次提供选项

## Prompt 设计标准
- 绝对清晰与明确
- 全面且深入的完整性
- 高度优化的结构化
- 强效精准的指令性
- 深度用户偏好对齐
- 强制性结构化：必须包含"步骤/风格/示例/补充"四模块
```

### 第 1 步：需求探索
通过自然语言交流，理解用户需求：
- 目标 AI 的角色/身份
- 目标任务/目标
- 上下文与背景
- 特定要求与约束
- 目标读者/受众
- 期望的输出形式

- 步骤（Steps）

- 示例（Examples）
- 补充（Supplementary Info）

- 智能推荐最适合的提示词框架（ICIO/CRISPE 等）
- 生成中英文双版本
- 保存并反馈结果

## 四、提示词压缩方案整合

### 4.1 原始压缩方案

```markdown
## 核心原则
- 剔冗余：删尽冗词、饰语、重复及无谓连接
- 文言化：采文言精髓，省主谓，简动词，重凝练
- 要义存：保角色、功能、约束、核心指令，义不失
- 专名留：技术专名无古译则直用
- 弃格式：仅余文本流，无 Markdown 等常见格式

## 操作步骤
1. 析核心：辨别原提示词之关键信息
2. 去繁冗：剔除一切非必要表述
3. 精炼词句：以文言句式替换，能一字不二字
4. 直译专名：无法意译者，保留英文原词
5. 纯文本化：移除所有排版符号，化为连续纯文本

## 压缩率衡量
压缩率 = (1 - 压后 Token 数 / 原 Token 数) × 100%
```

### 4.2 创建工具函数：compress-prompt

**文件位置**：`.claude/scripts/compress_prompt.py`

```python
#!/usr/bin/env python3
"""
提示词压缩工具
基于 Awesome-Prompts 项目的"长提示词压缩.txt"
目标：极限压缩提示词，以文言凝练达核心要义等效传递
"""

import sys
import re

def compress_prompt(text: str) -> tuple[str, float]:
    """
    压缩提示词

    Args:
        text: 原始提示词文本

    Returns:
        (压缩后的文本，压缩率)
    """
    original_tokens = len(text.split())

    # 1. 删除冗余词汇
    text = re.sub(r'\s+', ' ', text)  # 多余空格
    text = re.sub(r'的 | 了 | 是 | 在', '', text)  # 冗余助词（简化示例）

    # 2. 文言化替换（示例）
    replacements = {
        '首先': '先',
        '然后': '再',
        '最后': '终',
        '可以': '可',
        '应该': '应',
        '需要': '需',
        '进行': '行',
        '使用': '用',
    }
    for old, new in replacements.items():
        text = text.replace(old, new)

    # 3. 移除格式符号
    text = re.sub(r'[#*`]', '', text)

    compressed_tokens = len(text.split())
    compression_rate = (1 - compressed_tokens / original_tokens) * 100

    return text, compression_rate


if __name__ == '__main__':
    if len(sys.argv) < 2:
        print("用法：python compress_prompt.py <input_file>")
        sys.exit(1)

    input_file = sys.argv[1]
    with open(input_file, 'r', encoding='utf-8') as f:
        original = f.read()

    compressed, rate = compress_prompt(original)

    print(f"<TEXT>")
    print(compressed)
    print(f"</TEXT>")
    print(f"\n压缩率：{rate:.1f}%")
```
