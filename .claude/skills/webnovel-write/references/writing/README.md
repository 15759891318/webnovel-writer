## 快速参考卡片

### 开篇黄金三章检查清单
```
□ 第 1 章前 500 字是否有异常事件？
□ 第 1 章结束是否有悬念钩子？
□ 第 3 章是否建立核心冲突？
□ 金手指是否在 3 章内初现？
□ 世界观是否分层释放？
□ 主角是否主动做决定？
□ 配角是否有辨识度？
□ 每章结束是否有钩子？
```

### 战斗描写检查清单
```
□ 战斗前是否有敌我分析？
□ 是否有战前准备环节？
□ 敌人是否有智商（非无脑）？
□ 是否有能力克制/策略？
□ 是否有底牌层出？
□ 是否有反转？
□ 战后是否有代价描写？
□ 战斗节奏是否符合模板？
□ 爽点是否合理分布？
□ 招式命名是否简洁？
```

### 剧情设计检查清单
```
□ 主线是否清晰？
□ 伏笔是否有铺垫？
□ 伏笔是否有回收？
□ 悬念是否合理分布？
□ 转折是否意料之外情理之中？
□ 多线切换是否流畅？
□ 节奏是否张弛有度？
□ 每章是否有推进？
□ 每 10 章是否有小高潮？
□ 每卷是否有大高潮？
```

### 人物塑造检查清单
```
□ 主角是否有清晰欲望？
□ 主角是否有明显缺陷？
□ 主角性格是否一致？
□ 配角是否有辨识度？
□ 配角是否有独立故事？
□ 反派是否有智商？
□ 情感线是否有铺垫？
□ 人物成长是否可见？
□ 群像是否各有特色？
□ 镜像反派是否设计好？
```

### 爽点设计检查清单
```
□ 每 1-2 章是否有小爽点？
□ 每 5-10 章是否有中爽点？
□ 每 30-50 章是否有大爽点？
□ 爽点前是否有铺垫？
□ 爽点后是否有他人震惊？
□ 收获是否明确展示？
□ 反派是否有智商？
□ 主角是否靠实力获胜？
□ 爽点密度是否合适？
□ 是否有先抑后扬？
```

### 节奏把控检查清单
```
□ 全书节奏是否符合四卷结构？
□ 每卷是否有明确节奏变化？
□ 是否张弛有度？
□ 章末是否有钩子？
□ 章首是否承接自然？
□ 多线切换是否流畅？
□ 最近 5 章是否有爽点？
□ 最近 10 章是否有高潮？
□ 是否连续紧张超过 5 章？
□ 下一章是否应该放松？
```

### 对话写作检查清单
```
□ 这段对话是否有功能？（信息/性格/剧情/情感）

□ 不同角色是否能区分？
□ 对话节奏是否适合场景？
□ 是否有废话可以删掉？
```

## 核心技巧速查

### 开篇方式选择
```
穿越苏醒式 → 《诡秘之主》《我不是戏神》
封闭空间式 → 《十日终焉》《诡舍》
日常异常式 → 《斩神》《序列：吃神者》
失忆寻名式 → 《我不是戏神》
```

### 战斗类型选择
```
能力对抗型 → 《诡秘之主》《序列》
智斗博弈型 → 《十日终焉》《诡舍》
团队配合型 → 《斩神》
绝境求生型 → 《诡舍》《我不是戏神》
```

### 爽点类型选择
```
升级爽 → 序列晋升、等级提升
信息差爽 → 轮回记忆、穿越知识
智斗爽 → 规则破解、推理破局
守护爽 → 保护家人朋友
身份爽 → 多重身份掉马
反转爽 → 绝境翻盘
```

### 节奏模板
```
单章节奏：3000 字
0-500：压抑 → 500-1500：准备 → 1500-2500：爆发 → 2500-3000：余韵

十章节奏：3 万字
1-3 章：铺垫 → 4-6 章：发展 → 7-9 章：高潮 → 10 章：休整

百章节奏：30 万字
1-30 章：第一单元 → 31-60 章：第二单元 → 61-90 章：第三单元 → 91-100 章：卷高潮
```

## 创作方案参考

已生成两个完整创作方案，详见 `material/创作方案/` 目录：

### 方案一：《旧日序列》
- **题材**：克苏鲁 + 序列途径 + 都市脑洞
- **核心设定**：月瞳降临，序列觉醒
- **金手指**：污染值锁定（免疫代价）
- **组织**：异常事务局（加入）
- **主题**：守护 vs 疯狂

### 方案二：《规则废土》
- **题材**：废土末日 + 都市异能 + 诡异怪谈
- **核心设定**：规则怪谈降临，90% 人类消失
- **金手指**：规则之眼（看穿/修改规则）
- **组织**：希望基地（建立）
- **主题**：秩序 vs 混乱

## 使用指南

### 写作前
1. 查看 [opening-techniques.md](opening-techniques.md) 确定开篇方式
2. 查看 [character-design.md](character-design.md) 设计主角
3. 查看 [supporting-characters.md](supporting-characters.md) 设计配角群像
4. 查看 [world-building.md](world-building.md) 设计世界观和能力体系
5. 查看 [plot-design.md](plot-design.md) 设计主线和伏笔

### 写作中
1. 查看 [combat-techniques.md](combat-techniques.md) 写战斗场景
2. 查看 [dialogue-advanced.md](dialogue-advanced.md) 写对话场景
3. 查看 [cool-point-design.md](cool-point-design.md) 设计爽点
4. 查看 [pacing-control.md](pacing-control.md) 把控节奏

### 写作后
1. 使用各文档的检查清单自查
2. 检查爽点密度是否合理
3. 检查节奏是否张弛有度
4. 检查伏笔是否有铺垫和回收
