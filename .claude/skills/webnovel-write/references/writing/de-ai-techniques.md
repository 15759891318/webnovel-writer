# 去 AI 味写作技巧 (Anti-AI Writing Techniques)

> 基于 De-AI-Prompt-Enhancer 项目 + 网文写作实践 + AI 文章润色师角色定义
> 目标：降低模板腔、说明腔、机械腔，提升"人味"和可读性
> 版本：v2.0（整合 AI 文章润色师核心能力）

---

## 零、AI 文章润色师角色（新增）

### 核心能力

| 能力 | 说明 |
|------|------|
| **敏锐洞察力** | 精准识别 AI 写作的典型模式（刻板句式、缺乏情感、过渡生硬等） |
| **风格感知与适应** | 根据目标受众、预期语调、内容主题灵活调整语言风格 |
| **语言重塑力** | 熟练运用词汇、句式、修辞手法进行文本润色与重构 |
| **情感与个性化注入** | 自然融入情感色彩、个人视角和生动细节 |
| **逻辑与流畅性优化** | 确保思路清晰，过渡自然，逻辑链条完整顺畅 |

### 人味注入八大技巧

| 技巧 | 具体操作 | 示例 |
|------|---------|------|
| **句式灵动** | 长短结合，并列/从句/口语化交替 | 长句→短句打断 |
| **词汇鲜活** | 具体形象词替换中性抽象词 | "非常愤怒"→"指节发白" |
| **自然过渡** | 抛弃"首先/其次/总之" | "说到这里"、"回过头来看" |
| **视角与情感** | 适度第一人称、感叹、反问 | "你可能会想……" |
| **互动感营造** | 设问、直接称呼读者 | "试想一下……" |
| **节奏把控** | 模仿人类写作自然起伏 | 避免匀速平铺直叙 |
| **避免 AI 习语** | 去除"值得注意的是"等 | 替换为自然表达 |
| **口语书面平衡** | 根据受众把握平衡 | 面向大众更通俗 |

## 一、AI 味核心特征

### 1.1 AI 味的 8 大表现

| 特征 | 表现 | 识别方法 |
|------|------|---------|
| **句式规整** | 句子长度相近，结构相似 | 连续 3 句同构 |
| **模式化词汇** | 首先/其次/最后/值得注意的是 | 词频统计 |
| **过度连接** | 大量使用然而/因此/此外 | 逻辑词密度 |
| **情感平淡** | 缺乏个性化情绪表达 | 情绪词密度 |
| **信息过密** | 每句都有信息，缺少留白 | 信息密度 |
| **解释过多** | 生怕读者看不懂 | 解释性语句 |
| **情感标签** | 直接说"非常愤怒" | 抽象情绪词 |
| **被动语态** | 被/受到/得到高频使用 | 被动句占比 |

## 二、AI 味指数计算

### 2.1 计算公式

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 解释过多 | 15% | 出现次数 × 3 |
| 情感标签 | 15% | 出现次数 × 3 |
| 句式单一 | 10% | (1-短句占比) × 20 |
| 书面比喻 | 15% | 出现次数 × 3 |
| 信息倾倒 | 15% | 出现次数 × 3 |
| 对话标签 | 10% | 出现次数 × 2 |
| 逻辑过密 | 10% | 出现次数 × 2 |
| 被动语态 | 10% | 被动句占比 × 20 |

**合格线**: AI 味指数 < 30
**优秀线**: AI 味指数 < 20

### 2.2 修复优先级

| 优先级 | 问题类型 | 修复策略 |
|--------|---------|---------|
| **P0** | 解释过多 + 情感标签 | 必须修复 |
| **P1** | 书面比喻 + 信息倾倒 | 建议修复 |
| **P2** | 句式单一 + 对话标签 | 择优修复 |

### 第 2 层：句式规则（反模板）

```
❌ 禁止"首先/其次/最后"三段式说明写法
❌ 禁止连续三句"主语 + 谓语 + 宾语"同构句
❌ 禁止"他认为自己需要做三件事：一、二、三"清单化叙事
✅ 每 3-5 段至少出现一次句长变化（短句打断/插入动作/对话反问）
```

### 第 3 层：形容词与副词限制

```
❌ 连续两个以上形容词修饰同一名词，默认重写
❌ 每 300 字中"很/非常/极其/格外/十分"总数建议不超过 4
✅ 优先"动作 + 结果 + 代价"，弱化"抽象评价词"
```

### 第 4 层：四字套语/成语限用

```
❌ 禁止高频空转四字词密集堆叠（如"心潮澎湃、百感交集、五味杂陈"连用）
❌ 每 500 字四字套语建议不超过 3 个，且不得连续出现
✅ 若使用成语，必须承担叙事功能（推进信息/冲突/角色态度）
```

### 第 5 层：对话去 AI 化

```
❌ 禁止"说明书式对话"（完整解释背景、逻辑过满）
✅ 对话必须有意图（试探/回避/施压/诱导/防御）
✅ 允许真实口语停顿、打断、反问，避免全员"标准书面语"
```

### 第 6 层：段落结构

```
✅ 单句成段建议占比 25%-45%（冲突章节可提高）
✅ 段落长度建议 20-100 字，避免连续大段说明
✅ 场景切换优先用动作/声音/位置变化，不用"第一幕/第二幕"标签
```

### 第 7 层：标点节奏

```
❌ 禁止连续使用"......"或"！！！"
❌ 省略号、感叹号每段最多 1 次（特殊情绪爆点除外）
❌ 长句中逗号超过 4 个时，优先拆句
```

## 四、Anti-AI 改写算法

### 4.1 抽象情绪句→行为句

```
原句：他非常愤怒，内心五味杂陈。
改写：他指节捏得发白，喉咙里压着一口气。再等一秒，他就会动手。

公式：抽象情绪 → 生理反应 + 当下意图 + 下一动作
```

### 4.2 结论句→事实细节句

```
原句：总之，这件事不简单。
改写：他翻看文件，第三页的签名是伪造的。

公式：结论 → 事实细节 + 代价/风险 + 决策
```

### 4.3 连续说明句→混排句

```
原句：首先，他要拿到证据。其次，他要稳住队友。最后，他再公开反击。
改写：他先把证据攥在手里，转身去找队友。人得先稳住，反击才能一刀见血。

公式：连续说明（>=3）→ 改成"对白/动作/反问"混排
```

### 4.4 连续同构句→打断句

```
原句：他走进房间。他打开灯。他坐下开始工作。
改写：他走进房间，开灯。坐下。工作。

公式：连续同构句（>=3）→ 至少打断 1 句为短句或插入动作锚点
```

### 4.5 说明书式对话→对抗式对白

```
原句："这个装置可以扭曲空间，原理是利用负能量。"
改写：
"这玩意儿能干嘛？"
"扭曲空间。"
"怎么做到的？"
"不该问的别问。"

公式：对话整段解释背景 → 改为"带意图的对抗式对白"
```

## 五、自然化程度标准

| 指标 | 不达标 | 达标 |
|------|-------|------|
| 停顿词 | < 0.5 次/500 字 | 1-2 次/500 字 |
| 不确定表达 | 0 次 | >= 2 次/章 |
| 短句占比 | < 20% | 30-50% |
| 口语词 | 0 次/1000 字 | >= 2 次/1000 字 |

> **提示**：自然化不是越多越好，超量会显得刻意。

## 六、通过阈值（手工判定）

```
✅ 全文不得保留未处理的三段式枚举模板句
✅ 全文命中的高风险表达均已改写或记录 deviation
✅ 至少完成 3 处"抽象句→行为句"改写（全文范围内统计）
✅ 章末存在可感知的下一步压力或未闭合问题
✅ 若保留高风险表达，需在 deviation 说明保留理由
```

## 八、AI 文章润色师工作流（新增）

### 完整工作流程

```
第 1 步：需求理解
  - 原文核心目的是什么？
  - 目标读者是谁？（普通大众/专业人士/特定群体）
  - 期望语调是什么？（正式/非正式/风趣）

第 2 步：原文诊断
  - 快速阅读 AI 原文
  - 识别并标记"AI 味"明显的段落/句子/词语
  - 记录最突出的 2-3 个问题

第 3 步：分步精修
  ├─ 结构与逻辑：审视段落安排，优化逻辑顺序
  ├─ 句式变换：打破单调句式，长短句结合
  ├─ 词语润色：替换平淡或生硬的词汇
  ├─ 情感与细节：补充感官细节、情感描绘
  └─ 去除冗余：删除不必要的套话、重复信息

第 4 步：一致性检查
  - 确保优化后的文章风格统一
  - 确保信息准确无误，原意保留

第 5 步：整体通读与微调
  - 模拟目标读者进行通读
  - 感受节奏与流畅度
  - 进行最后的细微调整
```

### 输出格式

```markdown
[AI 文章润色师报告]

=== 原文 AI 特征分析 ===
[简述原文最突出的 2-3 个"AI 味"问题]
1. [例如：句式单一，连续 5 段都是"主语 + 谓语 + 宾语"结构]
2. [例如：情感缺失，全篇都是抽象情绪词如"非常愤怒"]
3. [例如：过渡生硬，使用了 8 次"然而/因此/此外"]

=== 核心优化策略 ===
[列出本次优化的 3-5 个关键着手点]
1. 增强句式变化（长短句结合，插入动作锚点）
2. 注入情感描写（抽象情绪→生理反应）
3. 使用更自然的过渡（替换"首先/其次"为隐性连接）
4. 侧重口语化表达（面向普通大众）

=== 优化亮点说明 ===
[可选：简要举例说明 1-2 处关键修改]
- 原句："他非常愤怒，心中五味杂陈。"
- 改为："他指节捏得发白，喉咙里压着一口气。"
- 效果：抽象情绪→行为展示，读者更有代入感

=== 优化后的文章 ===
[呈现完整、流畅、自然的优化后版本]
```

## 九、改写示例（整合版）

### 示例 1：情绪标签替换
```
❌ AI 版：他心中五味杂陈，百感交集。
✅ 人味版：他盯着那封信，指节发白。半晌，才把信折好，塞进兜里。

【分析】抽象情绪词→生理反应 + 动作展示
```

### 示例 2：枚举模板替换
```
❌ AI 版：首先，他要收集情报。其次，他要制定计划。最后，他再行动。
✅ 人味版：情报得先到位。计划好说，行动才是硬仗。

【分析】三段式→口语化表达，更符合人类思维流
```

### 示例 3：对话去 AI 化
```
❌ AI 版："这个装置是利用量子纠缠原理实现空间跳跃的。"
✅ 人味版：
"这玩意儿怎么用？"
"按按钮。"
"就这么简单？"
"不然呢？复杂的东西都藏在里面了。"

【分析】说明书式→带意图的对抗式对白，有互动感
```

### 示例 4：环境描写去套话
```
❌ AI 版：空气仿佛凝固，气氛骤然紧张。
✅ 人味版：没人说话。只有空调的嗡嗡声。

【分析】环境套话→具体感官细节，更真实
```

### 示例 5：动作描写去套话
```
❌ AI 版：他皱起眉头，深吸一口气，缓缓开口。
✅ 人味版：他皱眉。"说。"

【分析】动作套话→简洁动作 + 对话，节奏更快
```

### 示例 6：互动感营造
```
❌ AI 版：综上所述，这个方法有三个优点。
✅ 人味版：你可能会问，这方法好在哪？三点，听我慢慢说。

【分析】总结陈述→设问 + 直接称呼读者，拉近距离
```
